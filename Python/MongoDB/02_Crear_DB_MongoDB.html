¿Qué son Roles en MongoDB?

MongoDB utiliza un modelo de seguridad basado en roles, lo que significa que puede asignar roles específicos a usuarios específicos. Cada rol tiene un conjunto específico de privilegios que determinan lo que el usuario puede hacer. Este comando crea un nuevo usuario que tiene permiso para leer y escribir datos.

Los roles esperan una serie de elementos y estos elementos tienen que ser muy específicos, por lo que tendrán un formato de objeto, por lo que tendrán más llaves y esperarán un rol como clave y luego un conjunto de cadenas.

Rol Otorga privilegios para realizar lo especificado Comportamiento en Recurso. Cada privilegio se especifica explícitamente en el rol o heredado de otro rol o ambos.

clusterAdmin Proporciona el mayor acceso a la gestión de clústeres. Este rol combina la privilegios otorgados por el clusterManager, clusterMonitor, y hostManager roles. Además, el rol proporciona la dropDatabase acción.

read Proporciona la capacidad de leer datos en todas las colecciones non-system y el system.js Recopilación.

readWrite Proporciona todos los privilegios del read rol más capacidad para modificar datos en las colecciones non-system y system.js
